# Music Vault v1.0.0

Released 2026-07-13

## What Music Vault is

Music Vault is a standalone, local-first Windows music library and player. It
organizes and plays music already on the computer and can optionally turn an
authorized public or unlisted YouTube playlist into a persistent local library.
It has no Watchtower relationship or runtime dependency.

## Core V1 capabilities

- Local-file import, SQLite library persistence, custom playlists, cover art,
  search, album/artist browsing, and Windows playback
- Seek, persisted volume, autoplay, shuffle, repeat, and a FIFO manual queue
  that returns to its original playback context
- Optional public/unlisted YouTube synchronization with pagination, incremental
  identity reconciliation, structured failures, retry, and truthful outcomes
- Trusted manual metadata correction with provenance, protected values,
  history/undo, and explicit MusicBrainz candidate review
- Resumable existing-library remediation that analyzes before apply, leaves
  uncertain items unchanged, and can use verified MP3 backups/writeback and
  conflict-aware rollback
- Optional privacy-aware artist photos, neutral local App Status, centralized
  FFmpeg readiness, and non-admin desktop-shortcut support

## Installation

1. Download `MusicVault-v1.0.0-Windows-x64-Portable.zip` and its checksum from
   the [v1.0.0 GitHub Release](https://github.com/jcjohnson9/music-vault/releases/tag/v1.0.0).
2. Verify the ZIP's SHA-256 value.
3. Extract the complete portable folder to a writable location. Do not run the
   application from inside the ZIP.
4. Run `MusicVault.exe`.

The package needs neither the source repository nor a special working
directory. Windows SmartScreen may warn that the publisher is unknown because
v1.0.0 has no code-signing certificate. Music Vault does not request
administrator access and has no automatic updater.

## First launch and local-only use

A genuinely blank runtime opens a short first-run guide. Choose a writable data
location, optionally import one local music folder, and either complete or skip
online setup. Music Vault does not scan the whole computer.

Local import and playback work without a YouTube API key and without FFmpeg. An
existing established installation is recognized and is not reset or forced
through onboarding.

## Authorized YouTube synchronization

YouTube synchronization is optional. It supports public and unlisted playlists
through the YouTube Data API and anonymous yt-dlp extraction. It does not
support private-playlist OAuth and does not silently inspect browser cookies.
The user must acknowledge the authorized-use notice before configuring sync and
is responsible for using only music they own or are allowed to download.

An API key, when provided, is stored only in the local secret file. It is not
written to JSON configuration, App Status, logs, or the release package.

## FFmpeg

Synchronization conversion and some media processing require both
`ffmpeg.exe` and `ffprobe.exe`. Install or configure them separately through a
selected location, a portable tools location, or the system `PATH`. Music Vault
does not bundle or automatically download the FFmpeg command-line tools. Basic
startup, local import, and playback can continue without them.

The audio-quality choice controls the conversion target; it cannot add quality
that is absent from the source.

## Metadata correction and remediation

Manual edits remain local unless the user explicitly searches MusicBrainz.
Library remediation starts only on request and keeps analysis separate from
apply. Only strict, unique high-confidence matches are eligible for confirmed
automation; needs-review, ambiguous, no-match, locked, stale, and failed items
remain unchanged.

Optional supported MP3 tag writeback requires an additional explicit choice and
uses a verified full-file backup, temporary-copy mutation, tag readback, and
unchanged-audio-payload checks. Rollback refuses to overwrite a later conflict.
Unresolved items can continue to display legitimate embedded or YouTube-source
fallback metadata.

## Data location and backups

The default portable runtime directory is `data` beside `MusicVault.exe`. The
first-run guide can select a different writable location; Settings reports and
opens the active location. Runtime data can contain the database, playlists,
configuration, API key, App Status, downloads, artwork, reports, provider cache,
failures, archives, and backups.

Back up the complete private runtime data directory before moving an
installation or performing significant maintenance. Do not share an initialized
portable folder as though it were the clean release.

## Clean package boundary

The published portable ZIP contains no personal music, database, playlist,
configuration, API key, status, failed-item record, synchronization archive,
cover or artist-image cache, metadata report, remediation job, backup, personal
path, or personal screenshot. It also contains no bundled `ffmpeg.exe` or
`ffprobe.exe`. The application creates empty schema-v4 runtime data on first
launch.

## Known limitations

- Private-playlist OAuth is not supported.
- FFmpeg and ffprobe must be installed/configured separately for synchronization
  conversion and related processing.
- Unresolved metadata items may continue to use source or YouTube fallback
  values.
- Best Original audio format is not available yet.
- Multiple source playlists are not available yet.
- There is no Android version yet.
- There is no installer or automatic updater yet.
- The Windows executable is not code-signed.

These are planned boundaries, not failures of the supported V1 workflow.
